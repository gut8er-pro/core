## Failures

### BUG-001: Duplicate "Show missing information" toggle
- **Suite:** 06-vehicle, 07-condition
- **URL:** /reports/:id/details/vehicle, /reports/:id/details/condition
- **Report Type:** All
- **Severity:** Medium
- **Expected:** One "Show missing information" toggle (from layout)
- **Actual:** Two toggles visible — one from layout, one from the page component itself
- **Screenshot:** `testing/screenshots/06-hs-vehicle.png`, `testing/screenshots/07-hs-condition.png`
- **Root cause:** Both `details/layout.tsx` and the individual page (vehicle/condition) render a "Show missing" toggle. The layout one shows "29 sections need attention", the page one shows "16 fields need attention". Need to remove the page-level one or consolidate.

### BUG-002: PowerHP not persisted after reload
- **Suite:** 06-vehicle
- **URL:** /reports/:id/details/vehicle
- **Report Type:** All
- **Severity:** Low
- **Expected:** PowerHP auto-calculated from kW (110 kW → ~150 HP) and saved
- **Actual:** PowerHP field empty after reload. kW=110 persists but HP="" 
- **Root cause:** HP is likely auto-calculated client-side on kW blur but the calculated value isn't saved to the database via handleFieldBlur. The HP field may not trigger its own save.

### BUG-003: New Report dropdown difficult to interact with
- **Suite:** 02-dashboard
- **URL:** /dashboard
- **Severity:** Low (Playwright-specific, works manually)
- **Expected:** Click "New Report" → dropdown with 4 report types
- **Actual:** Dropdown toggles open/close inconsistently with Playwright clicks. Had to use API to create reports.
- **Note:** This may be a Playwright interaction issue, not a real bug. The dropdown uses `showReportTypeMenu` state toggle and `mousedown` outside click handler which can conflict with automated clicks.

### BUG-004: "50% Complete" and "30% Complete" are hardcoded
- **Suite:** 05-accident-info, 08-calculation
- **URL:** /reports/:id/details/accident-info, /reports/:id/details/calculation
- **Severity:** Medium
- **Expected:** Completion percentage reflects actual filled fields
- **Actual:** Accident Info always shows "50% Complete", Calculation always shows "30% Complete" regardless of data. The tab badges (0/6, 1/4 etc.) are dynamic and correct, but the in-page percentage is static.
- **Root cause:** Page components have hardcoded completion values that were never wired to dynamic calculation.
